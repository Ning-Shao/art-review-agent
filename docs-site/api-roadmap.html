后续 API 接入与 Roadmap

当前 Demo 不接真实 API。未来接入真实大模型时，应采用后端代理结构，避免 API key 暴露在浏览器端。

推荐技术结构

浏览器静态前端
  -> POST /api/reviews
后端代理 / Serverless Function
  -> 读取服务端环境变量中的 API key
  -> 校验、裁剪、脱敏前端输入
  -> 调用真实大模型 API
  -> 返回结构化评审结果
前端报告面板
  -> 渲染真实评审结果
  -> 写入历史记录

前端改造点

当前模块未来版本改造方向
src/services/aiReviewService.mock.js替换为调用 /api/reviews 的真实 service，或新增真实 service 后按环境切换。
src/data/mockCases.js保留为 Demo / fallback 数据，不再作为正式评分来源。
src/data/mockReport.js保留报告结构适配层，把 API 返回结果规范化为前端 report。
src/components/workForm.js增加 loading、错误态、重试和禁用状态。
src/components/fileUpload.js明确文件上传策略：元数据、base64、对象存储 URL 或后端处理文件。
src/components/chatPanel.js接入对话 API，并决定是否携带报告和图片上下文。

后端职责

  • 读取服务端 API key。
  • 校验作品标题、说明、项目类型、上传材料元数据等输入。
  • 控制文本长度、图片数量和单次请求成本。
  • 调用真实模型 API。
  • 返回稳定的结构化 JSON。
  • 处理错误、超时、重试和日志脱敏。
  • 做基础速率限制，避免 Demo 被滥用。

Roadmap

真实模型评审

新增后端代理，替换 Mock service，设计结构化评审结果 schema。

V0.4 未来版本

持久化与团队使用

增加身份、项目空间、完整报告保存、PDF 导出和检索管理能力。
